# Contributing to Quarc Starter Kit

Thank you for considering contributing to the Quarc Starter Kit!

## Coding Standards

- **PHP**: This project follows [PSR-12](https://www.php-fig.org/psr/psr-12/) coding standards.
- **Vue/JS**: We follow the standard Vue.js style guide and ESLint configuration included in the project.

## Development Workflow

1.  **Fork** the repository.
2.  **Clone** your fork locally.
3.  Create a new **branch** for your feature or fix (`git checkout -b feat/amazing-feature`).
4.  Make your changes.
5.  Run **tests** and linters (`npm run lint`, `php artisan test`).
6.  **Commit** your changes using [Conventional Commits](https://www.conventionalcommits.org/) (e.g., `feat: add new component`).
7.  **Push** to your fork and submit a **Pull Request**.

## Reporting Issues

If you discover a security vulnerability, please generally do not report it in public issues.

For general bugs, please open an issue on GitHub with a detailed description and reproduction steps.
